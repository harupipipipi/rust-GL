# Frequently Asked Questions

## General

### Why is the repository named "rust-GL" even though it does not use OpenGL?

Historical reasons. The project started as an OpenGL experiment, but the
rendering backend was replaced with
[softbuffer](https://crates.io/crates/softbuffer) (a pure CPU pixel buffer)
early in development. The name stuck. We are considering renaming the
repository — see [ROADMAP.md](ROADMAP.md) for details.

### Why softbuffer instead of wgpu / OpenGL / Vulkan?

softbuffer gives us **zero GPU dependencies**. The entire rendering path is
safe Rust with no shader code, no driver requirements, and no native graphics
API linkage. This means:

- The binary compiles on any platform with a C compiler (no GPU SDK needed).
- CI runs on headless servers without GPU drivers.
- The dependency tree stays small.

The trade-off is performance — every pixel is touched by the CPU. For small
UI tools and prototyping this is acceptable. A GPU-accelerated backend (wgpu)
is planned for the future; see [ROADMAP.md](ROADMAP.md).

### Can it display Japanese / Chinese / Korean text?

Yes, as long as a CJK font is available on the system. `FontManager` uses
[font-kit](https://crates.io/crates/font-kit) to discover system fonts and
builds a fallback chain. If the primary font lacks a glyph, the engine
searches the chain for a font that can render it. CJK word-wrap is also
implemented, so long strings break at appropriate character boundaries.

> **Note:** The fallback chain is not yet perfect — some edge cases may render
> a `□` placeholder. Fixes are tracked in the v0.3.0 milestone.

### Isn't winit 0.29 outdated?

We are aware. winit 0.29 was the stable release when the project started.
Migration to winit 0.30+ is listed in the [ROADMAP.md](ROADMAP.md) under
**Next (v0.3.0)**. The main blocker is the breaking API changes in winit's
event loop.

### Will this be published on crates.io?

Once the public API stabilises (likely around v0.4 or v0.5). Right now the
API changes on almost every commit, so publishing would create churn for
downstream users. You can depend on the Git repository directly in the
meantime:

```toml
[dependencies]
rust2d_ui = { git = "https://github.com/harupipipipi/rust-GL" }
```

### Was this code generated by AI?

The initial bootstrap was AI-assisted. Every line has since gone through human
review, and the project is iteratively refined by hand. See
[ARCHITECTURE.md](ARCHITECTURE.md) for the design rationale behind each
module — those decisions are human-driven.

## Technical

### What platforms are supported?

Anywhere winit + softbuffer run: **Linux** (X11 & Wayland), **macOS**, and
**Windows**. There is no web target and no mobile target at this time.

### How do I run the tests?

```bash
cargo test
```

Or use the convenience script that also checks formatting and lints:

```bash
./scripts/check.sh
```

### How do I contribute?

Open a pull request! Please run `./scripts/check.sh` before pushing to make
sure formatting, clippy, and tests all pass.
